One big issue I ran into a few times was due to the fact that MTG is a very very old game, there have been so many new mechanics and ideas, with the most problomatic ones being "dual faced cards" or cards that have different mechanis for different types of cards. I would often scan for result.artist, but it didn't have result.artist, it had result.card_faces[0].artist. Some had a single artist, some had 2 differnet artists, some had a single flavor text, some had 2 differnet flavor texts. It was very chaotic and I basically had to code in a check constantly if something is on a differnet face of the card. I just ignored every face but the first if it had multiple.

Requirements

My project 2 uses the ScryFall API to create a custom API URL through multiple differnet user inputs, requests that info from the API, and formats the returned info into a big list of boxes that show relevant information (in my case, just the info about the flavor, not the mechanics of the card). I will admit that my CSS had room for improvements, but I focused on functionality over astetics (somewhat ironically, considering the purpose of my sight is flavor over function). My page remembers the last selection searched, and fills that info in when the user loads the page again. My page creates a "next page" button when there is more info to load in, which loads in the next batch of scryfall results, and does not show that button if there are no more results. I was not able to code in a way to detect errors with the search unfortunately, so if someone searches for something with 0 results, scryfall returns an error, but I was not able to figure out how to detect that. I do have the functionality implimented for if it were to detect an error (at main.js line 259), Im just not sure how to detect an error.
